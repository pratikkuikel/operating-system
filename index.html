What is an Operating System?

Core Definition:

An operating system is a specialized software that:

  • Acts as an intermediary between hardware and user
  • Manages computer resources
  • Provides services to applications
  • Ensures system security and stability

Real-World Analogy:

Think of an operating system as a library manager who:

  • Organizes books (files) in shelves (storage)
  • Manages reading rooms (memory)
  • Handles book requests (process management)
  • Maintains library cards (user access)

Types of Operating Systems Based on Processing

Batch Processing

  • Similar jobs grouped together
  • Minimal user interaction
  • Example: Payroll processing
  • High throughput for similar tasks

Multi-programming

  • Multiple programs in memory
  • CPU always busy
  • Better resource utilization
  • Example: Modern PCs

Time-sharing

  • Multiple users share resources
  • Quick response time
  • Interactive computing
  • Example: Cloud servers

Real-time

  • Immediate response
  • Time-critical operations
  • Example: Air traffic control
  • Strict timing requirements

Common Misconceptions:

  • OS is not the same as BIOS
  • OS is not just a user interface
  • OS is not application software
  • More features don't always mean better performance

Important Terms to Remember:

  • Kernel: Core of the operating system

  • Process: Program in execution
  • Thread: Lightweight unit of processing
  • File System: Method of organizing data

Memory Management Techniques

Fixed Partitioning

  • Memory divided into fixed sizes
  • Simple implementation
  • Internal fragmentation
  • Limited flexibility

Dynamic Partitioning

  • Variable partition sizes
  • Better memory utilization
  • External fragmentation
  • More complex management

Paging

  • Fixed-size pages
  • No external fragmentation
  • Page table overhead
  • Hardware support needed

Segmentation

  • Variable-size segments
  • Logical division
  • Protection and sharing
  • External fragmentation possible

Virtual Memory Example:

When you run a large program on a computer with limited RAM:

  1. Program is divided into pages
  2. Frequently used pages stay in RAM
  3. Less used pages stored on disk
  4. Pages swapped as needed

Process States:

Process Scheduling Algorithms in Detail:

First-Come, First-Served (FCFS)

  • Simplest scheduling algorithm
  • Non-preemptive
  • Example:
    Process Burst Time P1 24ms P2 3ms P3 3ms Average waiting time = (0 + 24 + 27)/3 = 17ms
  • Disadvantage: Convoy effect

Shortest Job First (SJF)

  • Optimal for minimizing average waiting time
  • Can be preemptive or non-preemptive
  • Example:
    Process Burst Time P1 6ms P2 8ms P3 3ms Execution order: P3, P1, P2
  • Challenge: Predicting burst time

Priority Scheduling

  • Each process assigned a priority
  • Higher priority processes run first
  • Can lead to starvation
  • Solution: Aging mechanism

Round Robin

  • Time quantum based
  • Fair allocation of CPU
  • Example:
    Time Quantum = 4ms Process Burst Time P1 20ms P2 12ms Cycles: P1→P2→P1→P2→P1→P2→P1

Context Switching

  • Definition: Process of saving and loading process state
  • Components saved:
    • CPU registers
    • Memory maps
    • I/O states
  • Overhead considerations:
    • Cache invalidation
    • Memory pipeline flush
    • Kernel mode transition

Contiguous Allocation

  • Sequential blocks
  • Fast access
  • External fragmentation
  • Size limitations

Linked Allocation

  • Pointer-based structure

  • Sequential access only

  • Direct access support
